# OpenGrall

[![License](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](LICENSE)
[![Python](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)
[![Status](https://img.shields.io/badge/status-alpha-orange.svg)]()

**Открытый фреймворк для воплощённого ИИ. Один мозг, много тел.**
*AI-powered robotics framework / Robot with ChatGPT / LLM Robot / Voice-controlled robot / Autonomous robot*

OpenGrall — это модульная платформа для создания автономных роботов, управляемых большими языковыми моделями (LLM) и визуальными моделями (VLM). Фреймворк спроектирован для работы на слабом железе (смартфоны, Raspberry Pi), устойчив к деградации сенсоров и способен к эволюционному самообучению.

---

## 🧠 Философия (Почему OpenGrall устроен именно так)

*   **Модульность, а не монолит.** Добавление нового сенсора (лидар, ультразвук, ещё одна камера) или инструмента (поиск в интернете, управление манипулятором) не требует переписывания ядра. Система спроектирована как набор независимых блоков.
*   **Головной + спинной мозг (LLM + TinyML).** LLM принимает стратегические решения («поезжай на кухню»), TinyML на ESP32 обеспечивает быстрые рефлексы («объезд препятствия») с **абсолютным правом вето**. Безопасность гарантирована на уровне железа.
*   **Веса и доверие вместо принудительной синхронизации.** Мы не пытаемся заставить все сенсоры работать синхронно (как в ROS), потому что это приводит к «замираниям». В OpenGrall каждый сенсор работает в своём темпе, а `SensorMemory` и `WeightCalculator` сами решают, каким данным можно доверять прямо сейчас, основываясь на их возрасте, истории и наличии аномалий.
*   **Живучесть, а не стерильная точность.** Робот спроектирован для реального мира. Ему не нужна идеальная лаборатория. Он **сам** поймёт, что лидар загрязнился (вес упал), что камера ослеплена (вес упал), и переключится на те сенсоры, которые ещё работают. Система ориентируется на то, что есть, даже если это всего один ультразвуковой дальномер.
*   **Эволюция (экспериментально).** Робот учится на своих действиях, формирует стратегии и со временем перестаёт дёргать LLM для рутинных задач. Этот модуль — задел на будущее, открытый для экспериментов.

## 💡 Ключевые инновации (Что делает OpenGrall особенным)

### ⏳ Истинная асинхронность (против «заикания» ROS)
В классических системах (ROS) узлы синхронизируются по самому медленному. Если VLM «думает» секунду — робот **замирает** на секунду.
В OpenGrall каждый сенсор — это независимый поток данных. `SensorMemory` хранит **последнее валидное значение** с весом. Агент принимает решение **прямо сейчас**, на основе самых свежих данных, не дожидаясь отстающих. LLM может думать 500 мс — робот в это время **плавно едет по инерции**, управляемый TinyML.
**Результат:** Робот не «заикается». Он движется плавно и непрерывно, как живое существо.

### 📝 Семантическая компрессия реальности (LLM-friendly контекст)
LLM не нужны сырые данные. Ей нужен **смысл**.
`ContextBuilder` сжимает 500 000 точек лидара в одну строку: `lidar: front=0.4м, left=2.0м, right=2.5м | ближе 80см: 15°;45° ~40см, v=1.2м/с →`.
VLM описывает сцену: `сцена: коридор, человек 2.5м, путь: free`.
Всё это собирается в компактный, **взвешенный** промпт (с указанием возраста и надёжности данных). LLM получает не «шум», а **чистую семантическую картину мира**.
**Результат:** Даже слабая LLM (<1B) принимает адекватные решения, потому что ей не нужно продираться через терабайты сырых данных.

### ⚖️ Динамическое доверие: «Иммунная система» робота
Каждый сенсор имеет **вес**, который меняется в реальном времени.
- Лидар забрызган грязью? `WeightCalculator` видит аномалию (все сектора показывают одинаково) и **снижает его вес с 0.95 до 0.5**.
- VLM ослеплён солнцем? Его вес падает до 0.3.
- Данные устарели (возраст >2 сек)? Их вес экспоненциально **затухает**.

Агент **автоматически** опирается на те сенсоры, которым сейчас можно доверять. LLM получает готовые «подсказки» о надёжности: `[вес: 0.95, возраст: 0.02с]`.
**Результат:** Робот не впадает в ступор при откате одного сенсора. Он **адаптируется**, как живой организм, перераспределяя доверие.

## 🏗️ Архитектура

```

open_grall/
├── core/                    # Ядро: SensorMemory, WeightCalculator, StrategyLearner
├── sensors/                 # Сенсоры: Lidar, VLM, Аудио, Ультразвук, Одометрия
├── memory/                  # Память: Диалоги, Маршруты, Эпизоды
├── vision/                  # Зрение: VLM-клиент, Визуальная память
├── orchestration/           # Инструменты: 16+ действий (движение, речь, поиск)
├── agents/                  # LLM-клиенты: Ollama (Vikhr), YandexGPT
├── config.py                # Единый конфигурационный файл
├── agent_v5.py              # Главный класс робота
├── server_v7.py             # WebSocket сервер (нервная система)
└── dashboard.html           # Дашборд отладки (FREEZE, трассировка LLM)

```

## 📦 Что нужно для запуска (Hardware Requirements)

**Система спроектирована так, чтобы работать с любым набором сенсоров.**
*   **Абсолютный минимум:** Смартфон с Android 10+ (или ПК), ESP32, 1-2 мотора. Робот сможет ездить по командам и реагировать на голос.
*   **Рекомендуемые модули (для автономности):**
    *   **Лидар** (например, от пылесоса Xiaomi) — для точной геометрии.
    *   **Камера** (любая, доступная через браузер) — для визуального понимания (VLM).
    *   **Микрофон + динамик** — для голосового общения.

## 🚀 Быстрый старт

1.  **Склонируйте репозиторий:**
    ```bash
    git clone https://github.com/Ferum93/OpenGrall.git
    cd OpenGrall
    ```

2.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

3.  **Настройте `config.py`:**
    *   Укажите модель LLM (рекомендуется Vikhr-2.0-1.5B).
    *   Настройте пути к Vosk, камере, серверу.

4.  **Запустите сервер (на смартфоне/ПК):**
    ```bash
    python server_v7.py
    ```

5.  **Запустите агента:**
    ```bash
    python agent_v5.py
    ```

6.  **Подключите ESP32** с прошивкой TinyML (прошивка в разработке, можно использовать заглушку).
