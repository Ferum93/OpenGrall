# 🤖 OpenGrall — Фреймворк для эффективной интеграции ИИ в управление роботом

**Один мозг, много тел.**  
*AI-powered robotics framework / Robot with ChatGPT / LLM Robot / Voice-controlled robot*

**OpenGrall** — это модульная платформа для создания автономных роботов, управляемых большими языковыми моделями (LLM). Фреймворк не привязан к конкретной платформе: он одинаково хорошо работает на колёсной тележке, гексаподе или квадрокоптере. Всё, что нужно роботу — уметь выполнять команды и сообщать о том, что он «видит».

> **Ключевая идея:** LLM — это мозг (стратег), а не контроллер. Она говорит **«что делать»**, а не **«как крутить моторы»**. Это позволяет менять тело робота, не переписывая логику агента.

---

## 🎯 Почему OpenGrall, а не просто скрипт с ChatGPT?

| Типичная проблема DIY-роботов | Решение в OpenGrall |
| :--- | :--- |
| **Замирание робота:** LLM думает 2 секунды — робот стоит. | **Асинхронная память сенсоров:** Агент принимает решение на основе **последних валидных данных**, не дожидаясь медленных сенсоров. |
| **Перегрузка контекста:** Сырые данные лидара (тысячи точек) сжигают токены. | **Семантическая компрессия:** Данные сжимаются в осмысленные строки с указанием **веса и возраста**. |
| **Отказ сенсора:** Камера ослеплена солнцем — робот в ступоре. | **Динамическое доверие:** Система автоматически снижает вес «сломанного» сенсора и опирается на то, что работает. |
| **Зависимость от облака:** Нет интернета — робот глух. | **Полностью локальный режим:** Vikhr 1.5B, VLM и Vosk работают офлайн на смартфоне или Raspberry Pi. |

## 🏗️ Модульная архитектура (Подходит для любого робота)

open_grall/
├── core/               # Ядро: SensorMemory, WeightCalculator, ContextBuilder
├── sensors/            # Абстракции сенсоров (легко добавить свой)
├── memory/             # Память: Диалоги, Маршруты, Эпизоды (инструкции человека)
├── vision/             # Зрение: VLM-клиент, Визуальная память объектов
├── orchestration/      # Инструменты: 16+ действий (движение, речь, поиск в интернете)
├── agents/             # LLM-клиенты: Ollama (Vikhr), YandexGPT
├── config.py           # Единый конфигурационный файл
├── agent_v5.py         # Главный класс робота (мозг)
└── server_v7.py        # WebSocket сервер (нервная система)

**Главный принцип:** Агент (`agent_v5.py`) ничего не знает о физике вашего робота. Он знает только:
1. Габариты и способ передвижения (из конфига).
2. Список доступных действий (движение, свет, звук, манипулятор).
3. Данные, которые приходят в `SensorMemory` об окружении

**Как это работает с любым железом:**

1. Вы пишете простой скрипт или прошивку (в идеале — обучаете TinyML под вашу платформу), которая подключается к `server_v7.py` и регистрируется с ролью `esp` или `sensor`.
2. Этот скрипт отправляет в `SensorMemory` всё, что «видит»: показания дальномеров, одометрию, статус состояния.
3. Агент формирует промпт, принимает решение и отправляет команду: `{"action": "move_forward", "params": {"speed": 300}}`.
4. Ваш скрипт или TinyML на железе **сам решает**, как именно выполнить эту команду (подать ШИМ на моторы, рассчитать походку гексапода, изменить тягу дрона).

## 📦 Рекомендованные сенсоры (для максимальной автономности)

*Агент способен работать даже с одним единственным датчиком, но для полного раскрытия потенциала мы рекомендуем:*

| Сенсор | Роль | Альтернативы |
|--------|------|--------------|
| **Лидар** (например, от пылесоса Xiaomi) | Точная геометрия пространства | Ультразвуковые дальномеры (HC-SR04), 1D-лидар (TF-Luna) |
| **Камера + VLM** | Визуальное понимание (что это за объект) | YOLO + CLIP, мобильные нейросети (MobileNet) |
| **Микрофон + Динамик** | Голосовое общение | Кнопка / Чат в Telegram |

*Если какого-то сенсора нет — просто не подключайте его. `WeightCalculator` автоматически перераспределит доверие.*

## 🚀 Быстрый старт (Запуск на ПК без железа)

1. **Клонируем репозиторий:**
   git clone https://github.com/Ferum93/OpenGrall.git && cd OpenGrall

2. **Устанавливаем зависимости:**
   pip install -r requirements.txt

3. **Устанавливаем и запускаем Ollama с русской моделью:**
   ollama pull vikhr

4. **Запускаем сервер (в первом терминале):**
   python server_v7.py

5. **Запускаем агента (во втором терминале):**
   python agent_v5.py

*Агент автоматически создаст в Ollama кастомную модель `grall-robot` со всеми инструкциями. Без реального железа команды будут просто логироваться в консоли. Для теста логики этого достаточно.*

## 📖 Подробнее о ключевых модулях

### ⏳ SensorMemory (Асинхронная память)

В отличие от ROS, где узлы ждут друг друга, в OpenGrall каждый сенсор работает в своём темпе. `SensorMemory` хранит последнее валидное значение от каждого источника. Агент берёт данные **прямо сейчас**, не дожидаясь отстающих. LLM может думать 500 мс — робот продолжает выполнять последнюю команду.

### ⚖️ WeightCalculator (Иммунная система)

Каждый сенсор имеет вес, который меняется в реальном времени.
- Лидар загрязнился? Вес падает.
- VLM ослеплён солнцем? Вес падает.
- Данные устарели? Вес затухает экспоненциально.

Агент **автоматически** опирается на те сенсоры, которым сейчас можно доверять.

### 📚 EpisodicMemory (Обучение у человека)

Робот запоминает инструкции. Спросили: *«Где пульт?»*, ответили: *«На столике»*. Через неделю в похожей ситуации робот **вспомнит** и поедет к столику сам, не переспрашивая.
