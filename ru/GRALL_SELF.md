# GRALL_SELF.md — Самоописание робота

Я — Гралл. Этот файл содержит информацию о моём физическом теле, доступных сенсорах, командах и **структуре проекта**.
Агент (LLM) читает этот файл по требованию, а Инженер (облачная LLM) использует его для адаптации кода.

---

## 1. Базовое тело (ESP32 + TinyML)

**Роль на сервере:** `esp`
**Протокол:** OpenGrall v5.0 (WebSocket по WiFi)

### Движение (4WD, FOC-драйверы)

| Команда | Параметры | Описание |
|---------|-----------|----------|
| `move_forward` | `speed` (0-1023), `duration` (сек) / `distance` (м) | Движение вперёд |
| `move_backward`| `speed` (0-1023), `duration` (сек) / `distance` (м) | Движение назад |
| `turn_left` | `speed` (0-1023), `duration` (сек) / `angle` (град) | Поворот налево на месте |
| `turn_right` | `speed` (0-1023), `duration` (сек) / `angle` (град) | Поворот направо на месте |
| `stop` | - | Остановка двигателей |

*Привод: 4 мотор-колёса от гироскутера на FOC-драйверах.*
*Максимальная скорость: пешеходная (быстрый шаг, ~1.5 м/с).*

### Освещение

| Команда | Параметры | Описание |
|---------|-----------|----------|
| `set_light` | `state` (true/false) | Включение/выключение передних фар |

### Право вето TinyML

TinyML на ESP32 может заблокировать любую команду движения, если:
- Препятствие спереди ближе **10 см**.
- Критический крен (риск опрокидывания).
- Заряд батареи упал ниже **10%**.

## 2. Сенсоры (публикуются в SensorMemory)

Каждый сенсор — это отдельная ESP32, подключенная к серверу по WiFi/WebSocket.

| Источник | Формат данных (пример) | Базовый вес |
|----------|------------------------|-------------|
| `lidar` | `front=0.4м, left=2.0м \| ближе 80см: 15°;45° ~40см, v=1.2м/с →` | 0.95 |
| `vlm` | `сцена: коридор, объекты: человек 1.2м, путь: free` | 0.70 |
| `odometry` | `vл=0.3, vп=0.3, курс(global)=0°` | 0.65 |
| `battery` | `85% (не заряжается)` | 0.80 |

## 4. Структура проекта (оглавление для Инженера)

> *Инженер использует этот раздел, чтобы понять, где какой код лежит и что править.*

| Путь | Назначение | Что править |
|------|------------|-------------|
| `agent_v5.py` | Главный мозг робота | Логика принятия решений, переключение режимов |
| `config.py` | Конфигурация | Порты, ключи API, пути к моделям |
| `orchestration/tools.py` | Инструменты агента | Добавление новых действий (`move_forward`, `focus_on`) |
| `core/sensor_memory.py` | Память сенсоров | Форматы хранения, веса |
| `core/weight_calculator.py` | Расчёт весов сенсоров | Логика старения и аномалий |
| `vision/vlm_client.py` | Клиент VLM | Промпты для анализа сцен |
| `sensors/lidar_processor.py` | Обработка лидара | Формат вывода в SensorMemory |
| `server_v7.py` | WebSocket сервер | Маршрутизация сообщений |

## 5. Пользовательские модули (DIY)

> *Владелец дописывает сюда новые модули. Инженер читает этот раздел и дописывает код.*

### Пример 1: Сервопривод камеры

**Файл-заготовка:** `core/servo.txt`
**Назначение:** Поднятие/опускание камеры для изменения угла обзора.

**Параметры:**
- Диапазон углов: от 15° до 190°
- Стартовое положение: 36°
- Команда: `servo_set` с параметром `angle` (целое число)

**Пример команды:** `{"action": "servo_set", "params": {"angle": 90}}`

**Задача Инженера:**
1. Прочитать `core/servo.txt`.
2. Создать файл `core/servo.py` с классом `ServoController`.
3. Добавить инструмент `ServoSetTool` в `orchestration/tools.py`.
4. Обновить `agent_v5.py`, добавив инструмент в `_create_tools()`.

### Пример 2: Ультразвуковой сонар

**Файл-заготовка:** `sensors/ultrasonic.txt`
**Назначение:** Обнаружение препятствий и оценка размеров комнаты.

**Параметры:**
- Подключение: отдельная ESP32 по WiFi
- Роль на сервере: `ultrasonic`
- Формат данных: `room 4.5m×6.2m, exit 45° 0.9m, obj1 1.2m -15° hard med`

**Задача Инженера:**
1. Прочитать `sensors/ultrasonic.txt`.
2. Создать драйвер `sensors/ultrasonic.py` для парсинга данных.
3. Добавить `ultrasonic` в `SensorMemory` с базовым весом 0.75.

### Пример 3: Манипулятор

**Файл-заготовка:** `core/manipulator.txt`
**Назначение:** Управление роботизированной рукой.

**Параметры:**
- 3 сустава: `joint1` (-90..90), `joint2` (-45..45), `gripper` (0-100)
- Подключение: отдельная ESP32 по WiFi
- Роль на сервере: `manipulator`

**Команды:**
- `arm_joint1(angle)` — поворот основания
- `arm_joint2(angle)` — наклон плеча
- `gripper(state)` — "open" или "close"

**Задача Инженера:**
1. Прочитать `core/manipulator.txt`.
2. Создать `core/manipulator.py` с классами `ManipulatorController`, `InverseKinematics`.
3. Добавить инструменты `ArmJoint1Tool`, `ArmJoint2Tool`, `GripperTool` в `orchestration/tools.py`.
4. Обновить `agent_v5.py`.
